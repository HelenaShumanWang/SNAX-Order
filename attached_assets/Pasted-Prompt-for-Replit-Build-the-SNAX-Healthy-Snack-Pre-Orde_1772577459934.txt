Prompt for Replit: Build the "SNAX" Healthy Snack Pre-Order App

1. Project Overview

Build a mobile-first web application called SNAX, based on the concept "Future Me Ordered This." The app is a behavioral intervention designed to help busy college students make healthier late-night food choices. It works by allowing them to pre-order healthy snacks in the morning — when their self-control is higher — for delivery later that night.

The core of the app is a pre-commitment and accountability system. It combines a simple e-commerce flow (browsing, ordering, payment) with unique behavioral features like consumption tracking ("eat-rate"), streak counters, gamification rewards, and social accountability nudges.

Please build this as a full-stack web app (mobile-responsive) using React for the frontend and Node.js/Express for the backend, with a PostgreSQL database. Use Tailwind CSS for styling.

4. Database Schema

Please create the following tables:

SQL


-- Users table
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  phone_number VARCHAR(20),
  password_hash VARCHAR(255) NOT NULL,
  dietary_preferences JSONB DEFAULT '[]',
  streak_count INTEGER DEFAULT 0,
  avg_eat_rate DECIMAL(5,2) DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Menu items table
CREATE TABLE items (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  description TEXT,
  price DECIMAL(10,2) NOT NULL,
  image_url VARCHAR(255),
  category VARCHAR(50),
  nutritional_info JSONB,
  is_available BOOLEAN DEFAULT TRUE
);

-- Orders table
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  status VARCHAR(30) DEFAULT 'placed',
  delivery_address TEXT,
  delivery_time TIMESTAMP,
  total_cost DECIMAL(10,2),
  pledge_confirmed BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Order items junction table
CREATE TABLE order_items (
  id SERIAL PRIMARY KEY,
  order_id INTEGER REFERENCES orders(id),
  item_id INTEGER REFERENCES items(id),
  quantity INTEGER DEFAULT 1
);

-- Eat-rate confirmations
CREATE TABLE eat_rate_confirmations (
  id SERIAL PRIMARY KEY,
  order_id INTEGER REFERENCES orders(id),
  user_id INTEGER REFERENCES users(id),
  percentage_eaten INTEGER CHECK (percentage_eaten BETWEEN 0 AND 100),
  confirmed_at TIMESTAMP DEFAULT NOW()
);

-- Friends table
CREATE TABLE friends (
  id SERIAL PRIMARY KEY,
  user_id_1 INTEGER REFERENCES users(id),
  user_id_2 INTEGER REFERENCES users(id),
  status VARCHAR(20) DEFAULT 'pending',
  created_at TIMESTAMP DEFAULT NOW()
);

-- Feedback table
CREATE TABLE feedback (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id),
  order_id INTEGER REFERENCES orders(id),
  rating INTEGER CHECK (rating BETWEEN 1 AND 5),
  comment TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);






5. API Endpoints

Please create a RESTful API with these endpoints:

Authentication:

•
POST /api/auth/signup — Create a new user.

•
POST /api/auth/login — Authenticate and return a JWT.

Menu Items:

•
GET /api/items — Get all available snack items (with optional filters).

•
GET /api/items/:id — Get details for a single item.

Cart (client-side state is fine for MVP, or use a cart table):

•
GET /api/cart — Get current user's cart.

•
POST /api/cart — Add an item to the cart.

•
PUT /api/cart/:itemId — Update item quantity.

•
DELETE /api/cart/:itemId — Remove an item.

Orders:

•
POST /api/orders — Place a new order (checkout).

•
GET /api/orders — Get current user's order history.

•
GET /api/orders/:id — Get a specific order's details and status.

Eat-Rate & Streaks:

•
POST /api/orders/:id/confirm-eat-rate — Submit eat-rate for an order.

•
GET /api/profile/streak — Get current streak info.

Profile:

•
GET /api/profile — Get user profile data.

•
PUT /api/profile — Update profile (dietary preferences, etc.).

Friends & Social:

•
GET /api/friends — Get friends list with their stats.

•
POST /api/friends/add — Send a friend request.

•
PUT /api/friends/:id/respond — Accept or reject a friend request.

•
POST /api/friends/:id/nudge — Send an accountability nudge.

Feedback:

•
POST /api/feedback — Submit feedback for an order.

7. Design & Styling Guidelines

•
Color palette: Use a fresh, health-oriented palette. Primary: teal/green (#0D9488). Accent: warm orange (#F97316). Background: light gray (#F9FAFB). Text: dark gray (#111827).

•
Typography: Clean, modern sans-serif (e.g., Inter or system fonts).

•
Layout: Mobile-first responsive design. Use a bottom navigation bar on mobile.

•
Tone: Friendly, encouraging, and supportive — never punitive.




8. Build Order (Suggested Phases)

To keep the process logical and smooth, please build the app in this order:

1.
Phase 1 — Foundation: Set up the project structure, database, and authentication (Sign Up, Log In, JWT middleware).

2.
Phase 2 — Core Commerce: Build the Catalog Page, Item Details, Cart, Checkout, and Confirmation screens. Seed the menu data.

3.
Phase 3 — Behavioral Loop: Implement the Tracker (eat-rate confirmation), Reward/Loss pages, streak tracking logic, and the Pledge at checkout.

4.
Phase 4 — Profile & Social: Build the User Profile, Friends system, accountability nudges, and the Feedback page.

5.
Phase 5 — Polish: Add the Home Page dashboard, health tips, push notification placeholders, and final UI/UX polish.
